## Base Installation

The `pyworkflow_agents` module requires `langchain-core` as its foundation. Install it via the `agents` extra:

```bash
pip install 'pyworkflow-engine[agents]'
```

This gives you access to `ChatModel`, `TokenUsageTracker`, message types, and all agent exceptions.

## Provider Extras

To use a specific LLM provider, install the corresponding extra. Each extra bundles `langchain-core` plus the provider's LangChain integration package.

<Tabs>
  <Tab title="Anthropic">
    ```bash
    pip install 'pyworkflow-engine[agents-anthropic]'
    ```

    Installs: `langchain-core` + `langchain-anthropic`

    ```python
    from langchain_anthropic import ChatAnthropic
    from pyworkflow_agents import ChatModel

    llm: ChatModel = ChatAnthropic(model="claude-sonnet-4-20250514")
    ```
  </Tab>
  <Tab title="OpenAI">
    ```bash
    pip install 'pyworkflow-engine[agents-openai]'
    ```

    Installs: `langchain-core` + `langchain-openai`

    ```python
    from langchain_openai import ChatOpenAI
    from pyworkflow_agents import ChatModel

    llm: ChatModel = ChatOpenAI(model="gpt-4o")
    ```
  </Tab>
  <Tab title="Google">

### Install All Providers

To install every supported provider at once:

```bash
pip install 'pyworkflow-engine[agents-all]'
```

| Extra | Packages Installed |
|-------|-------------------|
| `agents` | `langchain-core` |
| `agents-anthropic` | `langchain-core`, `langchain-anthropic` |
| `agents-openai` | `langchain-core`, `langchain-openai` |

## Verifying Your Installation

After installing, verify that everything is working:

```python
# Always works (no langchain-core needed)
from pyworkflow_agents import AgentError, ProviderError
print("Exceptions OK")

# Requires langchain-core
from pyworkflow_agents import ChatModel, TokenUsageTracker
print("Types + tracking OK")

# Requires provider package (e.g. langchain-anthropic)
from langchain_anthropic import ChatAnthropic
print("Anthropic provider OK")
```

<Warning>
  If `langchain-core` is not installed, importing `ChatModel` or `TokenUsageTracker` from `pyworkflow_agents` will silently skip â€” no error is raised. The exception classes (`AgentError`, `ProviderError`, `ProviderNotInstalledError`) are always available regardless of installed dependencies.
</Warning>

## What Gets Imported

The `pyworkflow_agents` module conditionally exports based on what's installed:

| Export | Available Without langchain-core | Available With langchain-core |
|--------|--------------------------------|-------------------------------|
| `AgentError` | Yes | Yes |
| `ProviderError` | Yes | Yes |
| `ProviderNotInstalledError` | Yes | Yes |
| `ChatModel` | No | Yes |
| `MessageList` | No | Yes |
| `AIMessage`, `HumanMessage`, `SystemMessage`, `ToolMessage` | No | Yes |
| `TokenUsage` | No | Yes |
| `TokenUsageTracker` | No | Yes |
